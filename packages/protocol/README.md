# @privex/protocol

Protobuf schemas - the single source of truth for every data structure that
crosses the wire or sits in storage. TypeScript types are generated by
`protobufjs-cli`; Rust types by `prost` (see `server/build.rs`).

All schemas use `proto3`, package `privex`. Binary data (keys, signatures,
ciphertext, MACs) is always `bytes`. Timestamps are `uint64` Unix seconds.
`px_[hex]` ids stay `string` (textual identifiers, not raw binary).

## Type notes

- **`SealedEnvelope`** - what the server sees: recipient id, an ephemeral pubkey
  to open the sender cert, the encrypted cert, and the ratchet ciphertext.
  Optional CSAM proof/commitment only when the payload carries an image.
- **`Content`** - the plaintext `oneof` that gets sealed-sender encrypted. Add a
  new message type here when adding a new payload kind.
- **`RatchetMessage`** - Double Ratchet header + body. Padding is applied to the
  plaintext before encryption and is intentionally not modeled.
- **`KeyBundle`** vs **`PreKeyBundle`** - the former is the full directory record
  (many OPKs); the latter is one fetch result (exactly one OPK, plus KT proof).
- **`WireMessage`** - top-level wrapper. `payload` oneof routes everything;
  control frames mirror the WebSocket protocol.

## Build & test

```sh
pnpm --filter @privex/protocol build   # generates src/proto.js + src/proto.d.ts
pnpm --filter @privex/protocol test    # TS encode/decode + shared wire vector
```

Rust side: `cargo build` in `server/` runs `build.rs` (prost), and
`cargo test` runs the matching encode/decode test. Both the TS test and the
Rust test assert the same `EXPECTED_HEX` vector - byte-identical output is the
interop guarantee.
